# NuCo-AutoML

NuCo-AutoML is an end-to-end Automated Machine Learning (AutoML) framework designed to handle multimodal datasets containing tabular, text, and image data. By integrating the common-sense reasoning and expert prior knowledge of Large Language Models (LLMs) into traditional AutoML pipelines, NuCo-AutoML achieves a fully automated, zero-shot machine learning workflow.

## 🌟 Core Modules

The NuCo-AutoML pipeline is orchestrated sequentially through five core stages:

1. **Semantic Input Preparation (SIP)** *(Code implementation: MI-LLM & AFE-LLM)*
   Automatically infers the data type (modality) of each column using LLM semantic analysis and executes adaptive data cleaning strategies based on the inferred modalities.

2. **Semantic-guided Automated Feature Engineering (SAFE)** *(Code implementation: SAFE / formerly CAAFE)*
   Targets numerical and categorical data to automatically generate new, semantically meaningful features by leveraging LLMs to write feature engineering code.

3. **Modality-wise Consensus and Fusion (MCF)** *(Code implementation: MS-LLM)*
   Selects the most appropriate model architectures for each modality and determines the optimal late-fusion strategies based on dataset characteristics and scale.

4. **Context-aware Hyperparameter Tuning (CHT)** *(Code implementation: PA-LLM)*
   Recommends a context-aware warm-start search space for hyperparameters to accelerate the subsequent Bayesian optimization process.

5. **Pipeline Assembly and Execution (PAE)** *(Code implementation: main.py execution phase)*
   Dynamically assembles the selected architectures, fusion strategies, and search spaces into an executable PyTorch training script (`pipeline.py`), and automatically executes it with advanced mechanisms like Early Stopping and Out-Of-Memory protection.

## 🚀 Getting Started

### 1. Prerequisites
Ensure you have Python 3.8+ installed along with the required dependencies (e.g., PyTorch, HuggingFace Transformers, Optuna, Pandas, Scikit-Learn, OpenAI).

### 2. API Key Configuration
You need to configure your LLM API keys before running the pipeline:
- Set your API key in `NuCo-AutoML/config/settings.py`
- Set your API key in the SAFE module located at `SAFE/caafe/caafe.py` (around line 157)

### 3. Running the Pipeline
You can run the entire end-to-end pipeline seamlessly with a single command. 

```bash
cd NuCo-AutoML
python main.py --dataset YOUR_DATASET_NAME --model gpt-4o --safe_iterations 5
```

#### Detailed Arguments:
- `--dataset`: **[Required]** The name of your dataset. The data must be placed in `NuCo-AutoML/data/<dataset_name>/` containing `train_split.csv` and `test_split.csv`.
- `--model`: The LLM model to use. Default is `gpt-4o`.
- `--iterations`: Number of few-shot samples provided to the SIP module for modality inference. Default is `10`.
- `--safe_iterations`: Number of iterations for the SAFE module to generate and evaluate new features. Default is `5`.
- `--target`: The name of the target/label column in your dataset. Default is `AdoptionSpeed`.
- `--preference`: User preference for model selection (e.g., `"Best Accuracy, GPU Available"`).
- `--seed`: Global random seed for reproducibility. Default is `42`.

## 📊 Outputs

All generated assets will be saved in the `NuCo-AutoML/output/<YOUR_DATASET_NAME>/` directory:
- `dataset_state.json`: Inferred modalities from the SIP module.
- `afe_plan.json`: Cleaning strategies generated by the SIP module.
- `train_afe.csv` / `test_afe.csv`: Processed datasets after SIP and SAFE stages.
- `model_config.json`: Selected model architectures and fusion strategies from the MCF module.
- `pipeline.py`: The final executable PyTorch training script assembled by the PAE module.
- `run.log`: The execution log of the training and optimization process.
